# Security Policy

If you discover a security issue in any GDGoC APU project, please do not open a public issue. Instead, report it privately to the maintainers.

**Report to:** gdgoc.apu@gmail.com

## Supported versions

Security fixes are prioritised for actively maintained projects in the [gdgoc-apu](https://github.com/gdgoc-apu) organisation. If you are unsure whether a project is still maintained, include that in your report.

## What to report

Please report:

- Authentication issues
- Exposed secrets or API keys
- Data leakage
- Broken access control
- Vulnerable dependencies
- Unsafe user input handling
- Any issue that could affect user privacy or platform reliability

## What to include

Please include:

- Project or repository name
- Clear description of the issue
- Steps to reproduce
- Screenshots or logs if helpful
- Possible impact
- Suggested fix, if available

## Maintainer response

Maintainers will review the report and prioritise fixes based on severity. Please give the team reasonable time to investigate before sharing details publicly.
