# Governance

GDGoC APU projects are maintained by student contributors and project maintainers.

## Roles

### Project Lead

Responsible for project direction, roadmap, release planning, and final decisions.

### Maintainers

Responsible for reviewing pull requests, managing issues, checking quality, and guiding contributors.

Maintainers belong to the GitHub team [@gdgoc-apu/maintainers](https://github.com/orgs/gdgoc-apu/teams/maintainers). See [MAINTAINERS.md](MAINTAINERS.md) for the current list and [setup guide](.github/MAINTAINER_TEAM_SETUP.md) for org owners.

### Contributors

Anyone who contributes code, documentation, design, event resources, testing, or feedback.

## Decision-making

Small changes can be approved by maintainers through pull requests.

Large changes should be discussed through issues before implementation. Examples of large changes:

- Architecture changes
- Database schema changes
- Authentication changes
- Major UI redesigns
- New event platform or member portal features
- Changes affecting user data or privacy

## Review standard

Pull requests should be reviewed for:

- Correctness
- Maintainability
- Security
- User experience
- Documentation
- Alignment with the project goal

## Project values

We prioritise:

- Learning by building
- Beginner-friendly contribution
- Practical features
- Clear documentation
- Respectful collaboration
