# Contributing to GDGoC APU Projects

Thank you for contributing to GDGoC APU.

This organisation is maintained by student volunteers, so we value clear communication, respectful collaboration, and practical contributions.

## How to contribute

You can contribute by:

- Reporting bugs
- Suggesting features
- Improving documentation
- Adding workshop or event resources
- Fixing UI issues
- Reviewing pull requests
- Helping test features

## Before you start

1. Check existing issues first.
2. Choose an issue or create a new one.
3. Wait for a maintainer to confirm if the task is large.
4. Create a new branch.
5. Open a pull request with a clear description.

## Branch naming

Use clear branch names:

```txt
feature/add-event-registration
fix/navbar-mobile-layout
docs/update-readme
content/add-workshop-slides
```

## Commit style

Use simple and readable commit messages:

```txt
feat: add event registration form
fix: correct workshop date display
docs: update contribution guide
content: add cloud study jam slides
```

## Pull request checklist

Before opening a pull request:

- [ ] My changes are related to an issue or clear task
- [ ] I tested the feature locally
- [ ] I checked for spelling or formatting issues
- [ ] I added screenshots for UI changes
- [ ] I explained what changed in the PR description

## Review process

A maintainer will review your pull request.

For most changes, at least one approval is needed before merging.

For larger changes, maintainers may ask for discussion first. See [GOVERNANCE.md](GOVERNANCE.md) for how decisions are made.

## Community standard

Please be kind, patient, and respectful. Many contributors are students who are still learning. Read our [Code of Conduct](CODE_OF_CONDUCT.md) before participating.
