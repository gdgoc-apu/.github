# GDGoC APU — Organization Defaults

Default community health files, issue and pull request templates, and organisation profile for [GDGoC APU](https://github.com/gdgoc-apu).

This is the special `.github` repository. GitHub applies these defaults to any repository in the organisation that does not define its own version of a file.

## What's in this repository

| File / folder | Purpose |
| --- | --- |
| [profile/README.md](profile/README.md) | Organisation profile shown on [github.com/gdgoc-apu](https://github.com/gdgoc-apu) |
| [CONTRIBUTING.md](CONTRIBUTING.md) | How to contribute to org projects |
| [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md) | Community behaviour expectations |
| [GOVERNANCE.md](GOVERNANCE.md) | Roles, decision-making, and review standards |
| [SECURITY.md](SECURITY.md) | How to report security issues privately |
| [SUPPORT.md](SUPPORT.md) | Where to get help |
| [.github/ISSUE_TEMPLATE/](.github/ISSUE_TEMPLATE/) | Default issue forms for org repositories |
| [.github/PULL_REQUEST_TEMPLATE.md](.github/PULL_REQUEST_TEMPLATE.md) | Default pull request checklist |
| [.github/CODEOWNERS](.github/CODEOWNERS) | Default code review assignments |
| [MAINTAINERS.md](MAINTAINERS.md) | Maintainer team and responsibilities |
| [.github/MAINTAINER_TEAM_SETUP.md](.github/MAINTAINER_TEAM_SETUP.md) | How org owners create the GitHub team |

## Override behaviour

If a repository in the organisation has its own community health file or its own `.github/ISSUE_TEMPLATE/` folder, that repository's version takes precedence over these defaults.
